Manual de Usuario – Sistema de Administración de Condominios VITA S.A.

1. Introducción

1.1. Propósito del Sistema: El Sistema de Gestión de Condominios VITA S.A. es una aplicación de escritorio diseñada para centralizar y optimizar las operaciones administrativas, financieras y de mantenimiento de un complejo residencial. Su objetivo es proporcionar una herramienta intuitiva y potente para la administración diaria.

2. Primeros Pasos

2.1. Requisitos del Sistema:

  • Java Development Kit (JDK) 21 o superior.
  • Librería FlatLaf 3.4.1 (ver archivo README.md).

2.2. Inicio de la Aplicación:

  1. Abra el proyecto en el IDE Apache NetBeans.
  2. En el panel "Projects", localice el archivo CondominioGUI.java.
  3. Haga clic derecho sobre el archivo y seleccione Run File.

3. Vista General de la Interfaz

  • Panel de Navegación (izquierda): Botones para módulos principales (Dashboard, Residentes, Tareas, etc.).
  • Barra Superior: Muestra el título de la sección activa.
  • Área de Contenido (centro): Espacio donde se muestran tablas, formularios y estadísticas.

4. Funcionalidades Principales

  • 4.1. Dashboard: Vista general con métricas:
    • Total de Residentes
    • Tareas Pendientes y Completadas
    • Ingresos Totales
  • 4.2. Gestión de Residentes: Administración de información de residentes.
    • Campos: ID, Nombre, Departamento, Teléfono, Deuda.
    • Acciones: Agregar, Eliminar, Enviar Avisos de Pago, Buscar por ID o Nombre.
  • 4.3. Gestión de Tareas: Seguimiento de actividades del condominio.
    • Campos: ID, Descripción, Depto., Urgencia, Fecha, Costo, Estado, Prerreq.
    • Acciones: Agregar Tarea, Marcar Completada, Buscar por Costo.
  • 4.4. Gestión de Pagos: Control financiero de residentes.
    • Campos: ID, Nombre, Departamento, Deuda.
    • Acciones: Registrar Pago Individual, Aplicar Cuota General ($800), Ver Historial de Pagos.
  • 4.5. Reportes Financieros: Generación y control de estados financieros.
    • Campos: ID, Título, Fecha, Ingresos, Egresos, Balance, Status.
    • Acciones: Generar Nuevo Reporte, Eliminar Seleccionado.
  • 4.6. Historial de Actividades: Registro cronológico de todas las acciones significativas realizadas por los usuarios (solo lectura).

5. Flujos de Trabajo Comunes

Cómo Gestionar un Adeudo Completo:

  1. Aplicar cuota general desde el panel de Pagos.
  2. Enviar aviso de pago desde el panel de Residentes.
  3. Registrar pago individual con ID del residente y monto.
  4. Verificar el estado de la deuda en la tabla de Pagos.
  5. Consultar Historial para confirmar registros de cuota y pago.

Manual Técnico – Sistema de Administración de Condominios VITA S.A.

1. Introducción

1.1. Propósito del Documento: Proporcionar una descripción técnica detallada de la arquitectura, diseño y componentes del sistema. Dirigido a desarrolladores, personal técnico y evaluadores.

1.2. Alcance del Sistema: Solución integral para administrar un condominio con las siguientes funciones:

  • Gestión de Residentes (altas, bajas, búsquedas).
  • Gestión de Tareas de Mantenimiento, Seguridad y Administración.
  • Sistema de Priorización y Dependencias de Tareas.
  • Gestión Financiera (pagos y cuotas).
  • Generación de Reportes Financieros.
  • Historial de Actividades (auditoría).

1.3. Tecnologías Utilizadas:

  • Lenguaje: Java (JDK 23).
  • GUI: Java Swing.
  • Look and Feel: FlatLaf.
  • IDE: Apache NetBeans.
  • Control de Versiones: Git/GitHub.

2. Arquitectura del Software

  • Modelo: Clases de datos (Residente, Tarea, Pago, Reporte).
  • Vista: Clases gráficas (CondominioGUI, ResidentesPanel, etc.).
  • Controlador: SistemaController, gestiona lógica de negocio y comunicación Modelo–Vista.

3. Estructuras de Datos y Algoritmos

  • Tablas Hash: HashMap para residentes/tareas. Acceso en O(1).
  • Colas de Prioridad: PriorityQueue en GestorTareas (urgencia/fecha).
  • Árboles Binarios: ArbolBinarioBusqueda para residentes por nombre.
  • Grafos: Dependencias de tareas mediante lista de prerrequisitos.
  • Pilas: Historial de actividades en orden LIFO.
  • Recursividad y Búsqueda Binaria: Inserción y búsqueda en árboles + algoritmo "Divide y Vencerás" para tareas por costo.

4. Estructura del Proyecto

GUI: CondominioGUI.java, InicioPanel.java, ResidentesPanel.java, TareasPanel.java, ReportesPanel.java, PagosPanel.java, HistorialPanel.java, RoundedPanel.java, Theme.java.

Controlador: SistemaController.java.

Modelos: Residente.java, Tarea.java, Urgencia.java, Reporte.java, Pago.java, Actividad.java.

Estructuras: ArbolBinarioBusqueda.java, GestorTareas.java, Pila.java, ListaEnlazada.java, Nodo.java.

5. Funciones Principales

  • Residentes: getResidentes(), getResidentesConDeuda(), agregarResidente(), eliminarResidente(), búsquedas por ID/nombre.
  • Tareas: agregarTarea(), marcarTareaComoCompletada() con validación de dependencias.
  • Búsqueda: busquedaBinariaRecursiva() para encontrar tareas por costo.
  • Pagos: registrarPago(), aplicarCuotaGeneral().
  • Reportes: generarReporteFinanciero(), generarAvisosDePagoFormateados().
  • Historial: getHistorialActividades() usando Pila LIFO.

6. Funciones en la Interfaz (Paneles)

  • actualizarTabla(): Sincroniza tablas con datos actuales.
  • styleTable(JTable): Aplica estilos visuales uniformes.
  • createStyledButton(): Genera botones estilizados con acciones asignadas.
  • Métodos de acción: registrarPago(), eliminarResidente(), generarReporte() abren diálogos (JOptionPane) y actualizan tablas.
